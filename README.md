 # Design Pattern Principle
 
 ## Strategy
 
 * 바뀌는 부분은 캡슐화 한다.
 
 * `inherit`보다는 `composition`을 활용한다.
 
 * 구현이 아닌 인터페이스에 맞춰서 프로그래밍한다.
